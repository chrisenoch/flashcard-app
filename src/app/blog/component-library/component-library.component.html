Component Library Prototype

Objectives

  • Use your component styles in both Angular and React.
  • Media query support that works with SSR.
  • Themes and nested themes that works with SSR.
  • Styles out-of-the-box but completely customisable.
  • Expose every layer of HTML for full customisation and to avoid problems with ngDeep.
  • Customise the styles and/or theme on-the-fly.
  • Make reusable custom components.
  • Support light and dark mode.
  • Compatible with Tailwind and CSS modules.

Basic example

<app-button buttonText="Example" size="sm" rounded="full" href="/some/address" variant="secondary" ></app-button>
  • The above button is a link because it specifies the href prop.
  • Link buttons simulate disabled state via data attributes.

SX prop

  • All levels of the HTMl are exposed. Add or remove any class with the sx prop.

Media queries

  • Add the Tailwind classes to the sx prop.

Customise the theme on-the-fly

  • We create a Theme context by passing the instance of the theme down to other components via prop-drilling.
  • Although not as convenient as React context or Angular services, this should help avoid hydration and UI mismatch issues caused by SSR.
